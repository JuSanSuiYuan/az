## 一句话总结

| 语言 | 定位 | 特点 |
|------|------|------|
| **C3** | 更好的C | 简单、实用、接近C |
| **Zig** | 简单而强大 | 独立、快速、comptime强大 |
| **AZ** | 现代系统语言 | LLVM/MLIR、完整工具链、模块化 |

## 核心差异

### 错误处理

```
C3:  Result类型 + ! 语法
Zig: 错误联合类型 + try/catch
AZ:  Result类型 + 显式检查
```

### 内存管理

```
C3:  手动管理
Zig: Allocator模式
AZ:  手动 + 可选GC
```

### 编译器

```
C3:  自研 → LLVM IR
Zig: 完全自研
AZ:  LLVM + MLIR
```

### 包管理

```
C3:  ❌ 无
Zig: ✅ 内置
AZ:  ✅ chim
```

## 相似度矩阵

```
        C3    Zig   AZ
C3      100%  30%   60%
Zig     30%   100%  40%
AZ      60%   40%   100%
```

### 解读

- **AZ与C3最相似 (60%)** - 借鉴了C3的错误处理和模块系统
- **Zig最独特** - 与其他两者相似度都较低
- **三者都是优秀的系统编程语言**

- ✅ 项目规模大

### 2. 模块化设计 ⭐⭐

### 3. 完整工具链 ⭐⭐⭐

- 包管理器

- 可选GC

## AZ的设计理念

```
AZ = C3的理念 + Zig的特性 + LLVM的技术 + 自己的创新
```

### 借鉴C3
- ✅ Result类型错误处理
- ✅ 模块系统设计
- ✅ 简洁的语法

### 借鉴Zig
- ✅ comptime编译时执行
- ✅ defer语句
- ✅ 包管理理念

### 基于LLVM
- ✅ MLIR多级IR
- ✅ 强大的优化
- ✅ 成熟的生态

### 自己的创新
- ✅ 可选的AZGC
- ✅ chim包管理器
- ✅ 完整的工具链
